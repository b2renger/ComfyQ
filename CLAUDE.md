**Federation — LAN fleet monitor (UDP beacon + standalone Electron app) [implemented 2026-06-27; now packaged + auto-updating (see the entry above) and entering user testing 2026-06-28; single-machine verified, two-machine test pending]** — first federation work. Built on branch `feature/federation-fleet-monitor`. Verified locally end-to-end: machine identity/GPU/RAM/IP detected, beacon→listener round-trip, `GET /federation/self` live on the running dev server, Electron app boots + renders. App is branded with the ComfyQ mark ([desktop/assets/icon.svg](desktop/assets/icon.svg) → 512px `icon.png` for the window/taskbar + electron-builder; inline SVG header logo in [desktop/renderer/index.html](desktop/renderer/index.html)). Realizes Phase-F F1+F2 via a **UDP multicast status beacon** + a **cross-platform Electron "fleet monitor"** in a new top-level `desktop/` workspace — deliberately **diverging from the originally-locked mDNS/in-browser-panel design** (owner decision; see [implementation_plan.md](implementation_plan.md) → Phase F "Implementation path chosen"). Each ComfyQ server **sends a JSON status snapshot every ~5 s** over UDP — to the multicast group `239.255.42.99:41999` **and** to each interface's directed/limited broadcast (built-in `dgram`, no transport dep). The dual send is deliberate: consumer Wi-Fi/multi-AP LANs frequently drop multicast and multi-homed machines pick the wrong default interface, so **directed broadcast is what actually makes same-subnet peers see each other** (multicast TTL is 4 to survive a router hop; receiver joins multicast on *all* interfaces, broadcast is received automatically when bound to `0.0.0.0`). Runs in **both admin and student mode** so even **idle admin rigs with nothing launched** appear; the desktop app **listens** (peer map + stale-expiry live in the app, no per-peer polling). **Note:** the lab LAN is a **/23** (`10.10.16.0`–`10.10.17.255`), so the directed broadcast is `10.10.17.255` — broadcast addr is computed from each iface's real netmask, not assumed /24. **Discovery on broadcast-blocked LANs (subnet auto-scan + static peers):** managed/school Wi-Fi (the lab's `lecolededesign.com` net, a /23 `10.10.16.0`–`10.10.17.255`) enables client isolation for broadcast+multicast, so UDP beacons never arrive even though unicast works fine (verified: ping + `curl http://<peer>:3000/federation/self` both succeed, UDP beacon doesn't). So the app discovers machines over **unicast** three ways, all merged into one peer map ([desktop/main.js](desktop/main.js)): (1) UDP beacons (`source:'beacon'`); (2) **subnet auto-scan** — sweeps the local subnet probing `http://<ip>:3000/federation/self` (concurrency 48, 1.5s timeout, every 60s; a /23 sweep ≈ 16s; `source:'scan'`); (3) **Add machine by IP** for other subnets (`source:'poll'`, persisted). Discovered + static hosts are fast-polled every 5s to stay fresh; auto-scan is on by default with a toggle + "Rescan now" and skips subnets > /20 (4096 hosts) to stay polite. All of it reuses the existing Express `:3000` (no new firewall rule — same path the web UI uses) and persists to `userData/fleet-config.json` (`{staticPeers, autoScan}`; seedable via `COMFYQ_FED_PEERS`). IPC: `get/add/remove-static-peer`, `rescan`, `set-auto-scan` ([desktop/preload.js](desktop/preload.js)). **On by default** (`config.federation.enabled:true`; a **Network presence** toggle in AdminConfig flips it **live** — beacon re-reads enabled state each tick + `runtime.beacon.kick()` sends one immediately on enable — via `PUT /admin/federation`) and **read-only** — the only action is a "Schedule a job" button that `shell.openExternal`s `http://<rig-ip>:5173`. ## Runtime / environment
- **Node: LTS only.** Currently **Node 24 "Krypton"**, pinned in [.nvmrc](.nvmrc) (`24`) with `engines.node` = `>=22.0.0 <25.0.0` in the root, [server](server/package.json), and [client](client/package.json) `package.json`. Odd-numbered "Current" releases (23, 25, …) are unsupported — the lab runs varied machines and needs an LTS line. When Node 26 becomes LTS (~Oct 2026), bump `.nvmrc` + the three `engines` ceilings.
- **Windows winget gotcha:** winget ships Node as two packages. `OpenJS.NodeJS` is the **wrong** (Current) one; use **`OpenJS.NodeJS.LTS`**. To swap: `winget uninstall --id OpenJS.NodeJS --exact` then `winget install --id OpenJS.NodeJS.LTS --exact` (needs admin; globals in `%AppData%\npm` survive).
- **`better-sqlite3` is the only native dep**, pinned to v12+ for N-API prebuilds (one binary across Node versions, no per-machine recompile). After any Node *major* change, wipe `node_modules` and `npm install` so the ABI matches.

## Serving model — plain HTTP (do NOT re-add HTTPS)
- The dev/workshop UI is served over **plain HTTP** (Vite on :5173, proxying the Express backend). The self-signed HTTPS layer (`@vitejs/plugin-basic-ssl`) was tried and **reverted 2026-05-19**: self-signed certs can't be trusted across BYOD Safari + Chrome + mobile without installing a CA on every device, and Safari refused to extend a click-through to the websocket and to media downloads.
- **Don't reintroduce `@vitejs/plugin-basic-ssl` or `https: true`** in [client/vite.config.js](client/vite.config.js). The only feature it enabled — in-browser webcam capture (`getUserMedia`, needs a secure context) — was removed; image/video inputs are uploaded as files (a phone's file picker still offers "Take Photo").
- **Secure-context APIs don't work off-localhost over plain HTTP** — anything gated on `window.isSecureContext` (`navigator.clipboard`, `getUserMedia`, …) is unavailable on `http://<lan-ip>`. For clipboard, use [client/src/utils/clipboard.js](client/src/utils/clipboard.js) `copyToClipboard()` (tries the async API, falls back to a hidden-`<textarea>` + `execCommand('copy')`); never call `navigator.clipboard.writeText` directly.

## Key implementation patterns

**Workflow registration:** Each workflow is a folder bundle: `workflows/<id>/<id>.api.json` (the ComfyUI API-format export) + `<id>.meta.json` (parameters, description, category). The `meta.json` is read-only; admins edit via UI (writes to `.config.meta.json` override file). This keeps the export distinct from deployment tweaks.

**Output detection (generic):** No `class_type` whitelist. [server/media/mediaTypes.js](server/media/mediaTypes.js) classifies by file extension: `.glb/.gltf` → model3d, `.spz/.splat/.ksplat` → splat, `.mp3/.wav/…` → audio, `.mp4/.webm/…` → video, else image. New output formats Just Work™ — no code changes needed.

**Subfolder serving:** ComfyUI save nodes write to subfolders (e.g., `SaveAudioMP3` → `audio/track.mp3`). The wire carries the full output-root-relative filename; [server/executor/jobExecutor.js](server/executor/jobExecutor.js) folds the subfolder into `filename` and clears the `subfolder` field so `/images/:filename(*)` serves them without breaking existing routes.

**Multi-output galleries:** When a job produces >1 image, the lightbox renders [ImageGallery.jsx](client/src/components/ui/ImageGallery.jsx) (main image + thumbnails + prev/next). When it has both a splat and a mesh, a toggle switches between the 3D viewers. Audio outputs render the AudioPlayer. The Scheduler grid shows a "{N} views" badge for multi-output jobs.

**Student param form is ONE shared component (`DynamicParamFields`):** [client/src/components/DynamicParamFields.jsx](client/src/components/DynamicParamFields.jsx) renders the exposed-parameter form from a `parameter_map` and is the single source of truth for how students see inputs. It's rendered by **both** [BookingDialog.jsx](client/src/components/BookingDialog.jsx) (the real booking, with live state) **and** the admin [WorkflowMetaEditor.jsx](client/src/components/admin/WorkflowMetaEditor.jsx) "Student preview" (throwaway state) — so the admin's preview can never drift from production. Props: `paramMap`, `values`, `onValueChange(key,value)`, `mediaPreviews`, `recalledMedia`, `mediaChangeHandler(key)→(file)→void`, `mediaRemoveHandler(key)→()→void`. The editor preview converts the editor's param-array (`nodeId`, `enabled`, …) into the wire `parameter_map` shape (key→{type,label,default,options,min,max,step,disabledWhen,required,field,order,enabled}); `field` is needed because seed detection (`isSeedParam`) keys off it. **If you add a new param *type*, add its branch here once** and both the booking form and the preview get it.

**Param field gating (`disabledWhen`):** A parameter can declare `disabledWhen: { param, equals }` in the meta to grey itself out when another param holds a value. Used for either/or prompts (raw vs. LLM-enhanced). The field still submits; the workflow's switch node ignores the unused branch. **Preserved through the editor (2026-06-26):** `edit-data` returns it and the editor's save payload re-sends it, so an admin open+save no longer drops a workflow's gating (there's still no UI to *edit* `disabledWhen` — it round-trips untouched).

**Mask-draw input (`mask` param type):** For inpainting, a param `type: "mask"` renders [MaskDrawField.jsx](client/src/components/capture/MaskDrawField.jsx) instead of an uploader: the user picks a base image and **brushes the region to replace**, and the component composites it into a single RGBA PNG where the painted area is **transparent** (`globalCompositeOperation='destination-out'`). ComfyUI's `LoadImage` derives `MASK = 1 − alpha`, so transparent → mask 1.0 → "inpaint here" — one file feeds both the image (out 0, RGB) and the mask (out 1). The produced File flows the **same** path as an image upload (`onChange`→`handleMediaChange`→`/upload`→generic materializer), so the server is untouched. `mask` is treated as `image` everywhere the value flows: it's in `MEDIA_TYPES` (BookingDialog), required-by-default + `inferType` (parser), `maxInputEdge` (meta editor), and calibration stages a plain image for it (no alpha → empty mask → no-op inpaint, still runs the sampler → valid timing). Brush colour is cosmetic only (export reads alpha; recolour is `source-in`). Adding the type means editing both the zod enum ([schemas.js](server/config/schemas.js)) **and** the duplicated `PARAM_TYPES` array in WorkflowMetaEditor — they aren't shared. **Mask polarity** is the standard ComfyUI convention (white = inpaint, the graph's `mask_invert:false` confirms); flip in `MaskDrawField.commit()` if a rig run inpaints the inverse.

**Text outputs (LLM / caption):** Nodes that emit text instead of a file (`PreviewAny` "Preview as Text", `ShowText`, …) report a `text`/`string` array in `history[promptId].outputs`. `_collectText` in [server/executor/outputCollector.js](server/executor/outputCollector.js) collects these as a `kind:'text'` output with the string **inline on the wire** (no file to serve); [server/executor/jobExecutor.js](server/executor/jobExecutor.js) forwards `o.text`, and `ImageLightbox`/`MediaPreview` render the caption. No `class_type` coupling — any text-emitting node Just Works, so image/video→text workflows (Gemma describe-it) land zero-code. First exercised by the Gemma 4 captioners.

**Friendly failure messages (`humanizeFailure`):** ComfyUI execution errors reach the user as the job's failure `reason`, computed at two sites — the WS `execution_error` event ([localComfyUIWorker.js](server/workers/localComfyUIWorker.js)) and the history-status fallback ([jobExecutor.js](server/executor/jobExecutor.js)); both carry the failing node's `class_type`. `humanizeFailure(message, nodeType)` in [errorMessages.js](server/executor/errorMessages.js) rewrites known signatures into actionable text (first case: no-face on a face-cropper node → "upload a clear photo with a single, fully visible face"), and **falls through to the raw message** for everything else (errors are clarified, never swallowed). Add new mappings here, not at the call sites. Keep guards tight (e.g. an OOM on a face node is *not* a no-face problem) to avoid mislabeling.

**Calibration (auto, single run split into load+gen, admin-mode):** [server/benchmark/benchmarkService.js](server/benchmark/benchmarkService.js) runs the workflow **once** (explicit `/free` first so models load) and splits the run at the first sampler `progress` event: start→first-step = `modelLoadSec`, first-step→end = `estimatedDurationSec`/`warmDurationSec` (the recurring/optimal cost the timeline uses); full wall time = `coldDurationSec` (first-run incl. load); plus `samplesPerSec` + `gpu`. **Do NOT add a second "warm" run** — ComfyUI caches identical graphs and returns the cached result in ~1s, which made calibration report 1s for a 25s job (regression fixed 2026-06-13). Instead `_randomizeSeeds()` bumps every literal `*seed` node field **and** `_buildCalibrationParams` randomizes every exposed seed param, so the single run always actually executes (cache-immune, even on re-calibration). Inputs **auto-resolved from `config.assets.dir`** (`D:\_assets`): `_resolveAsset(type)` picks a median image / smallest video / audio clip, staged via the worker's `InputUploader` (image falls back to a built-in PNG; missing video/audio asset → clear error). Gauge button works in **admin mode** via [server/benchmark/adminCalibrator.js](server/benchmark/adminCalibrator.js) (lazily spawns/attaches ComfyUI, reuses it, idle-frees VRAM, leaves ComfyUI running on shutdown). **No UI, no uploads.** Student-mode caveat: calibrate while the queue is idle (executor + calibration share the one worker). **Config freshness (2026-06-21):** `AdminCalibrator` reads ComfyUI paths **fresh from `configManager` on each launch/calibrate** (getters, not a boot-time snapshot), so a path saved in Settings is honored immediately by the gauge and the *Launch ComfyUI on the network* button; it restarts a ComfyUI it spawned when the saved paths change (don't reintroduce a constructor snapshot — that was the "launch ignores my new path" bug).

**Image upload guard:** Three-layer protection against OOM crashes from oversized images: (1) client throws `ImageProcessError` on HEIC/decode failure; (2) [MediaCaptureField.jsx](client/src/components/capture/MediaCaptureField.jsx) shows inline error; (3) [server/routes/uploads.js](server/routes/uploads.js) backstop rejects HEIC (extension + ftyp magic byte) and images not fitting 1920×1080 box in either orientation. Server uses `image-size` library (pure JS, no native dep).

**Light-theme color tokens:** Theme is class-based (`.dark`/`.light` on `<html>`) over CSS-var tokens in [client/src/index.css](client/src/index.css) (`@theme` → `--color-*`). A compat hack maps **`--color-white → --text`** so the ~150 existing `text-white` usages stay readable in light mode without a sweep. The gotcha: that makes `text-white`/`bg-white` on a **solid accent fill** (`bg-primary`, `bg-danger`, …) dark-on-dark in light mode. For text/icons/knobs on an accent fill use the **`--color-on-primary`** token (`text-on-primary`/`bg-on-primary`) — it stays light in both modes. New surfaces should prefer `bg-background`/`bg-surface` + `text-foreground`/`text-muted`, not hardcoded slate/black/white.

**Input retention & asset reuse:** Booked-job media uploads go to `<comfy>/input/comfyq_session__*` and the filename is stored in `job.paramValues` (so it's on the wire). The periodic `sweepStale()` ([inputUploader.js](server/workers/inputUploader.js)) only deletes the **`comfyq__`** (job-scoped, execution-time) prefix — **not** `comfyq_session__`, so the original uploads persist. "Use these settings" exploits this: [BookingDialog.jsx](client/src/components/BookingDialog.jsx) re-binds the recalled filename directly (no re-upload; the materializer injects it) and previews/plays it via `GET /input-media/:filename` ([uploads.js](server/routes/uploads.js), restricted to the `comfyq_` prefix + pinned inside the input dir).

**ASAP booking:** A booking with no slot picked sends `scheduledTime: null`; the server treats null-or-past as ASAP and schedules into the earliest non-colliding slot via `jobQueue.nextFreeSlot(now, durationMs)` (greedy walk over active jobs) instead of throwing a collision. An explicit future time keeps the normal `findCollisions` guard. `findReady` orders by `scheduled_at ASC, created_at ASC` (FIFO tiebreaker). The BookingDialog's segmented **ASAP / Pick-a-time** toggle drives `scheduledTime` (null vs a timestamp).

**Prompt display resilience:** `pickHeadlinePrompt()` in [BookingDialog.jsx](client/src/components/BookingDialog.jsx) finds the user's text at submit time (prefers textarea params, skips negative-keyed ones). `getDisplayPrompt()` in [jobDisplay.js](client/src/utils/jobDisplay.js) mines `job.params` at render time so historical jobs display even if exposed under non-`prompt` keys (e.g., LTX `positive_prompt`, Ideogram `description`).

## Federation (Phase F — F1+F2 shipped via beacon + packaged auto-updating desktop app, in user testing 2026-06-28; F3–F6 deferred)
The final v2 phase is multi-instance LAN federation. **The first slice shipped (2026-06-27→28)** as a **UDP multicast status beacon** + a standalone **Electron fleet monitor** (`desktop/`) — now packaged as a self-updating installer (see the "Desktop app — installers + GitHub auto-update" Latest-progress entry) — a deliberate divergence from the originally-locked mDNS/in-browser-panel design — see the Latest-progress entry above and [implementation_plan.md](implementation_plan.md) → "Phase F". Server code lives under [server/federation/](server/federation/); the beacon and `GET /federation/self` share one `buildSnapshot()`. **Conventions to keep:** beacon transport is built-in `dgram` UDP multicast (no mDNS dep added); federation is **on by default** but fully gated by `config.federation.enabled` so a disabled boot is byte-for-byte today's single-instance behavior; the new `config.instance`/`config.federation` schema blocks are **optional/defaulted** so existing `config.json` validates unchanged. The remaining locked design (mDNS, cluster-secret auth, cross-instance admin actions, AdminConfig panel, student station picker, orchestrator role) stays deferred. **Implication for other work:** don't bake in single-instance assumptions — if a feature surfaces state that should be aggregable across peers, expose it where `buildSnapshot()` / a future `/federation/*` route can return it without a schema migration.
