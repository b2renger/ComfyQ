## Serving model ‚Äî plain HTTP (do NOT re-add HTTPS)
- The dev/workshop UI is served over **plain HTTP** (Vite on :5173, proxying the Express backend). The self-signed HTTPS layer (`@vitejs/plugin-basic-ssl`) was tried and **reverted 2026-05-19**: self-signed certs can't be trusted across BYOD Safari + Chrome + mobile without installing a CA on every device, and Safari refused to extend a click-through to the websocket and to media downloads.
- **Don't reintroduce `@vitejs/plugin-basic-ssl` or `https: true`** in [client/vite.config.js](client/vite.config.js). The only feature it enabled ‚Äî in-browser webcam capture (`getUserMedia`, needs a secure context) ‚Äî was removed; image/video inputs are uploaded as files (a phone's file picker still offers "Take Photo").
- **Secure-context APIs don't work off-localhost over plain HTTP** ‚Äî anything gated on `window.isSecureContext` (`navigator.clipboard`, `getUserMedia`, ‚Ä¶) is unavailable on `http://<lan-ip>`. For clipboard, use [client/src/utils/clipboard.js](client/src/utils/clipboard.js) `copyToClipboard()` (tries the async API, falls back to a hidden-`<textarea>` + `execCommand('copy')`); never call `navigator.clipboard.writeText` directly.

## Key implementation patterns

**Workflow registration:** Each workflow is a folder bundle: `workflows/<id>/<id>.api.json` (the ComfyUI API-format export) + `<id>.meta.json` (parameters, description, category). The `meta.json` is read-only; admins edit via UI (writes to `.config.meta.json` override file). This keeps the export distinct from deployment tweaks.

**Output detection (generic):** No `class_type` whitelist. [server/media/mediaTypes.js](server/media/mediaTypes.js) classifies by file extension: `.glb/.gltf` ‚Üí model3d, `.spz/.splat/.ksplat` ‚Üí splat, `.mp3/.wav/‚Ä¶` ‚Üí audio, `.mp4/.webm/‚Ä¶` ‚Üí video, else image. New output formats Just Work‚Ñ¢ ‚Äî no code changes needed.

**Subfolder serving:** ComfyUI save nodes write to subfolders (e.g., `SaveAudioMP3` ‚Üí `audio/track.mp3`). The wire carries the full output-root-relative filename; [server/executor/jobExecutor.js](server/executor/jobExecutor.js) folds the subfolder into `filename` and clears the `subfolder` field so `/images/:filename(*)` serves them without breaking existing routes.

**Multi-output galleries:** When a job produces >1 image, the lightbox renders [ImageGallery.jsx](client/src/components/ui/ImageGallery.jsx) (main image + thumbnails + prev/next). When it has both a splat and a mesh, a toggle switches between the 3D viewers. Audio outputs render the AudioPlayer. The Scheduler grid shows a "{N} views" badge for multi-output jobs.

**Student param form is ONE shared component (`DynamicParamFields`):** [client/src/components/DynamicParamFields.jsx](client/src/components/DynamicParamFields.jsx) renders the exposed-parameter form from a `parameter_map` and is the single source of truth for how students see inputs. It's rendered by **both** [BookingDialog.jsx](client/src/components/BookingDialog.jsx) (the real booking, with live state) **and** the admin [WorkflowMetaEditor.jsx](client/src/components/admin/WorkflowMetaEditor.jsx) "Student preview" (throwaway state) ‚Äî so the admin's preview can never drift from production. Props: `paramMap`, `values`, `onValueChange(key,value)`, `mediaPreviews`, `recalledMedia`, `mediaChangeHandler(key)‚Üí(file)‚Üívoid`, `mediaRemoveHandler(key)‚Üí()‚Üívoid`. The editor preview converts the editor's param-array (`nodeId`, `enabled`, ‚Ä¶) into the wire `parameter_map` shape (key‚Üí{type,label,default,options,min,max,step,disabledWhen,required,field,order,enabled}); `field` is needed because seed detection (`isSeedParam`) keys off it. **If you add a new param *type*, add its branch here once** and both the booking form and the preview get it.

**Param field gating (`disabledWhen`):** A parameter can declare `disabledWhen: { param, equals }` in the meta to grey itself out when another param holds a value. Used for either/or prompts (raw vs. LLM-enhanced). The field still submits; the workflow's switch node ignores the unused branch. **Preserved through the editor (2026-06-26):** `edit-data` returns it and the editor's save payload re-sends it, so an admin open+save no longer drops a workflow's gating (there's still no UI to *edit* `disabledWhen` ‚Äî it round-trips untouched).

**Mask-draw input (`mask` param type):** For inpainting, a param `type: "mask"` renders [MaskDrawField.jsx](client/src/components/capture/MaskDrawField.jsx) instead of an uploader: the user picks a base image and **brushes the region to replace**, and the component composites it into a single RGBA PNG where the painted area is **transparent** (`globalCompositeOperation='destination-out'`). ComfyUI's `LoadImage` derives `MASK = 1 ‚àí alpha`, so transparent ‚Üí mask 1.0 ‚Üí "inpaint here" ‚Äî one file feeds both the image (out 0, RGB) and the mask (out 1). The produced File flows the **same** path as an image upload (`onChange`‚Üí`handleMediaChange`‚Üí`/upload`‚Üígeneric materializer), so the server is untouched. `mask` is treated as `image` everywhere the value flows: it's in `MEDIA_TYPES` (BookingDialog), required-by-default + `inferType` (parser), `maxInputEdge` (meta editor), and calibration stages a plain image for it (no alpha ‚Üí empty mask ‚Üí no-op inpaint, still runs the sampler ‚Üí valid timing). Brush colour is cosmetic only (export reads alpha; recolour is `source-in`). Adding the type means editing both the zod enum ([schemas.js](server/config/schemas.js)) **and** the duplicated `PARAM_TYPES` array in WorkflowMetaEditor ‚Äî they aren't shared. **Mask polarity** is the standard ComfyUI convention (white = inpaint, the graph's `mask_invert:false` confirms); flip in `MaskDrawField.commit()` if a rig run inpaints the inverse.

**Text outputs (LLM / caption):** Nodes that emit text instead of a file (`PreviewAny` "Preview as Text", `ShowText`, ‚Ä¶) report a `text`/`string` array in `history[promptId].outputs`. `_collectText` in [server/executor/outputCollector.js](server/executor/outputCollector.js) collects these as a `kind:'text'` output with the string **inline on the wire** (no file to serve); [server/executor/jobExecutor.js](server/executor/jobExecutor.js) forwards `o.text`, and `ImageLightbox`/`MediaPreview` render the caption. No `class_type` coupling ‚Äî any text-emitting node Just Works, so image/video‚Üítext workflows (Gemma describe-it) land zero-code. First exercised by the Gemma 4 captioners.

**Friendly failure messages (`humanizeFailure`):** ComfyUI execution errors reach the user as the job's failure `reason`, computed at two sites ‚Äî the WS `execution_error` event ([localComfyUIWorker.js](server/workers/localComfyUIWorker.js)) and the history-status fallback ([jobExecutor.js](server/executor/jobExecutor.js)); both carry the failing node's `class_type`. `humanizeFailure(message, nodeType)` in [errorMessages.js](server/executor/errorMessages.js) rewrites known signatures into actionable text (first case: no-face on a face-cropper node ‚Üí "upload a clear photo with a single, fully visible face"), and **falls through to the raw message** for everything else (errors are clarified, never swallowed). Add new mappings here, not at the call sites. Keep guards tight (e.g. an OOM on a face node is *not* a no-face problem) to avoid mislabeling.

**Calibration (auto, single run split into load+gen, admin-mode):** [server/benchmark/benchmarkService.js](server/benchmark/benchmarkService.js) runs the workflow **once** (explicit `/free` first so models load) and splits the run at the first sampler `progress` event: start‚Üífirst-step = `modelLoadSec`, first-step‚Üíend = `estimatedDurationSec`/`warmDurationSec` (the recurring/optimal cost the timeline uses); full wall time = `coldDurationSec` (first-run incl. load); plus `samplesPerSec` + `gpu`. **Do NOT add a second "warm" run** ‚Äî ComfyUI caches identical graphs and returns the cached result in ~1s, which made calibration report 1s for a 25s job (regression fixed 2026-06-13). Instead `_randomizeSeeds()` bumps every literal `*seed` node field **and** `_buildCalibrationParams` randomizes every exposed seed param, so the single run always actually executes (cache-immune, even on re-calibration). Inputs **auto-resolved from `config.assets.dir`** (`D:\_assets`): `_resolveAsset(type)` picks a median image / smallest video / audio clip, staged via the worker's `InputUploader` (image falls back to a built-in PNG; missing video/audio asset ‚Üí clear error). Gauge button works in **admin mode** via [server/benchmark/adminCalibrator.js](server/benchmark/adminCalibrator.js) (lazily spawns/attaches ComfyUI, reuses it, idle-frees VRAM, leaves ComfyUI running on shutdown). **No UI, no uploads.** Student-mode caveat: calibrate while the queue is idle (executor + calibration share the one worker). **Config freshness (2026-06-21):** `AdminCalibrator` reads ComfyUI paths **fresh from `configManager` on each launch/calibrate** (getters, not a boot-time snapshot), so a path saved in Settings is honored immediately by the gauge and the *Launch ComfyUI on the network* button; it restarts a ComfyUI it spawned when the saved paths change (don't reintroduce a constructor snapshot ‚Äî that was the "launch ignores my new path" bug).

**Image upload guard:** Three-layer protection against OOM crashes from oversized images: (1) client throws `ImageProcessError` on HEIC/decode failure; (2) [MediaCaptureField.jsx](client/src/components/capture/MediaCaptureField.jsx) shows inline error; (3) [server/routes/uploads.js](server/routes/uploads.js) backstop rejects HEIC (extension + ftyp magic byte) and images not fitting 1920√ó1080 box in either orientation. Server uses `image-size` library (pure JS, no native dep).

**Light-theme color tokens:** Theme is class-based (`.dark`/`.light` on `<html>`) over CSS-var tokens in [client/src/index.css](client/src/index.css) (`@theme` ‚Üí `--color-*`). A compat hack maps **`--color-white ‚Üí --text`** so the ~150 existing `text-white` usages stay readable in light mode without a sweep. The gotcha: that makes `text-white`/`bg-white` on a **solid accent fill** (`bg-primary`, `bg-danger`, ‚Ä¶) dark-on-dark in light mode. For text/icons/knobs on an accent fill use the **`--color-on-primary`** token (`text-on-primary`/`bg-on-primary`) ‚Äî it stays light in both modes. New surfaces should prefer `bg-background`/`bg-surface` + `text-foreground`/`text-muted`, not hardcoded slate/black/white.

**Input retention & asset reuse:** Booked-job media uploads go to `<comfy>/input/comfyq_session__*` and the filename is stored in `job.paramValues` (so it's on the wire). The periodic `sweepStale()` ([inputUploader.js](server/workers/inputUploader.js)) only deletes the **`comfyq__`** (job-scoped, execution-time) prefix ‚Äî **not** `comfyq_session__`, so the original uploads persist. "Use these settings" exploits this: [BookingDialog.jsx](client/src/components/BookingDialog.jsx) re-binds the recalled filename directly (no re-upload; the materializer injects it) and previews/plays it via `GET /input-media/:filename` ([uploads.js](server/routes/uploads.js), restricted to the `comfyq_` prefix + pinned inside the input dir).

**ASAP booking:** A booking with no slot picked sends `scheduledTime: null`; the server treats null-or-past as ASAP and schedules into the earliest non-colliding slot via `jobQueue.nextFreeSlot(now, durationMs)` (greedy walk over active jobs) instead of throwing a collision. An explicit future time keeps the normal `findCollisions` guard. `findReady` orders by `scheduled_at ASC, created_at ASC` (FIFO tiebreaker). The BookingDialog's segmented **ASAP / Pick-a-time** toggle drives `scheduledTime` (null vs a timestamp).

**Prompt display resilience:** `pickHeadlinePrompt()` in [BookingDialog.jsx](client/src/components/BookingDialog.jsx) finds the user's text at submit time (prefers textarea params, skips negative-keyed ones). `getDisplayPrompt()` in [jobDisplay.js](client/src/utils/jobDisplay.js) mines `job.params` at render time so historical jobs display even if exposed under non-`prompt` keys (e.g., LTX `positive_prompt`, Ideogram `description`).

## Federation (Phase F ‚Äî F1+F2 in progress via beacon; F3‚ÄìF6 deferred)
The final v2 phase is multi-instance LAN federation. **The first slice is being built now (2026-06-27)** as a **UDP multicast status beacon** + a standalone **Electron fleet monitor** (`desktop/`), a deliberate divergence from the originally-locked mDNS/in-browser-panel design ‚Äî see the Latest-progress entry above and [implementation_plan.md](implementation_plan.md) ‚Üí "Phase F". Server code lives under [server/federation/](server/federation/); the beacon and `GET /federation/self` share one `buildSnapshot()`. **Conventions to keep:** beacon transport is built-in `dgram` UDP multicast (no mDNS dep added); federation is **on by default** but fully gated by `config.federation.enabled` so a disabled boot is byte-for-byte today's single-instance behavior; the new `config.instance`/`config.federation` schema blocks are **optional/defaulted** so existing `config.json` validates unchanged. The remaining locked design (mDNS, cluster-secret auth, cross-instance admin actions, AdminConfig panel, student station picker, orchestrator role) stays deferred. **Implication for other work:** don't bake in single-instance assumptions ‚Äî if a feature surfaces state that should be aggregable across peers, expose it where `buildSnapshot()` / a future `/federation/*` route can return it without a schema migration.
